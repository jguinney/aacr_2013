Introduction

The 2013 AACR Annual Meeting provides a unique opportunity for biomedical researchers across a wide-array of disciplines to discover the latest in cancer research around KRAS and strengthen the community links with other researchers. Like most meetings, rudimentary tools are provided such as abstracts and ribbons to identify attendees based on the importance of their presentation at the meeting (general poster discussion, mini-symposium, oral presentation, awards, faculty). However, there is no systematic way for researchers to discover novel social connections premised on shared research interests or needs. While online social platforms like LinkedIn and ResearchGate have the potential to promote new social interactions among researchers, their shortcomings include lack of research specificity, and no opportunities for direct social interactions that a conference provides.

To facilitate a more transparent and robust description of the RAS research community, we propose the construction of a social interactome for AACR RAS researchers. Our two primary aims are :

  1. To promote discovery of fellow researchers engaged in similar research, and thereby encourage collaboration
  2. Provide a descriptive map of the types of research within the RAS community

Source Data

The social interactome is based on all RAS-related abstracts accepted for the 2012 and 2013 AACR annual meetings. We identified 1388 abstracts that fit this criteria (793 2013 abstract and 595 2012 abstracts) representing the work of 8813 scientists.

Data Analysis

Abstract text analysis

We tokenized all words in the RAS related abstracts to create a document-term matrix. We applied standard text mining normalization to this data matrix, including removal of white space, punctuation, and stop words. We also applied stemming algorithms to find word roots. Finally, we removed words that appeared infrequently (<2 abstracts).


Base interactome

We generated an abstract similarity matrix by first normalizing the word count matrix into a term frequency-inverse document frequency (tf-idf) matrix, and finally computing a document similarity matrix using the cosine distance metric.


Exemplar networks

We used the Affinity Message Passing algorithm (ref) to generate the global network view. This is essentially a hierarchical clustering algorithm that clusters similar nodes, and then identifies an "exemplar" for that cluster. These exemplars are then clusters, and so on. The colors of this graphs represent the different levels of clustering, from blue - green - red - yellow, representing the local to global clusters, respectively.


User Interface

Control Panel

The left-side panel contains a text field with auto-completion is provided for users to identify themselves or colleagues by name. Hit Enter or click Get network to load a network.

By default, a first-degree neighbor network is loaded showing n neighbors based on the users selection. Alternatively, the user can choose to see one of the prepared overview networks (see Data Analysis section above). Selecting a node in one network view will carry over to other network views, if that same author/abstract node is present. This allows the user to identify themselves (or a colleague) in each of the network views. If an authors of an abstract replied to our survey, then the node within the graph will have a red border.


Network Panel

To the right of the Control Panel, is an interactive network viewer powered by Cytoscape Web [ref]. After loading a network via the Control Panel, you will see nodes representing individual abstracts. Each node is labeled with first and last author names and mouse-over action will reveal the abstract title. Click on a node to see the complete abstract details in the panel below. The connections between nodes are based on the similarity scores from the abstract text analysis described above. The network is interactive, so you can also drag nodes to rearrange the view. Right-click on a node to access the context menu where you can select to load a new network extending from the selected node.


Abstract Panel

At the bottom of the page is a static text area that displays the complete abstract details per selection in the network above. If the survey was answered by one or more of the coauthors, those results will be displayed below the abstract.
